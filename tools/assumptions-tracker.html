Assumptions Tracker

Every idea is a stack of assumptions. Write them down, score them by uncertainty and impact, design the cheapest experiment that clears each one — and always test the riskiest assumption first. Data is saved locally in your browser.

What is this & when to use it ▾

The Assumptions Tracker turns every belief your idea depends on into a testable hypothesis, scored by impact × uncertainty so you test the riskiest first. For each you plan an experiment, a pass/fail threshold and a cost — making validation a managed pipeline, not a guess.

Use it when

You have assumptions to de-risk (desirability, feasibility, viability) and want to prioritise experiments by risk and manage validation cost across a project.

Don't use it when

You have nothing to test yet — surface assumptions first via the Value Proposition Canvas, Press Release or Opportunity Solution Tree.

How it fits the toolbox

It is the spine of the toolbox: pains, press-release tricky issues and OST experiments all flow in here as assumptions to validate.

How to use this & what each column means

Work in two phases. In Frame & prioritise, list every assumption, tag its risk type, and score it by uncertainty × impact — then tick the few worth testing. In Test, design the cheapest experiment for each, set the pass/fail threshold, record what you measured, and mark the result. The flow of a single assumption: belief → score → experiment → threshold → measured → result.

Risk categories — the kind of risk each assumption carries
Desirability
Does the customer actually want it? Demand and value risk.
Feasibility
Can we build and operate it? Technical and operational risk.
Viability
Does it sustain itself financially and organisationally? Economic risk.
Adaptability
Does it hold up as regulation, market or tech change over time?
Other
Anything that doesn't fit the four above.
Frame & prioritise columns
Assumption
What must be true, as a statement: "We believe that…". One per row.
Value
The value you expect if it holds (e.g. "> 60% adoption") — the belief, not yet tested.
Uncertainty
How sure you are it's true. Low = strong evidence; High = mostly a guess.
Impact
How badly the idea breaks if it's false. Low = minor; High = fatal.
Risk
Uncertainty × Impact (max 9). Test the highest first.
Evidence
Strength of proof: Opinion → Stated preference → Observed behaviour.
Test columns
Experiment
The cheapest, fastest test that could change your mind.
Threshold
The pass/fail bar, set before running — the number it must hit.
Measured
What the test actually recorded. Compare to threshold, then set result.
Cost (€)
What the experiment costs to run; selected ones are totalled.
Validate?
Tick the ones you'll test — the Test view shows only these.
Result
Pending, Confirmed (met threshold) or Refuted (didn't — still cleared knowledge).
